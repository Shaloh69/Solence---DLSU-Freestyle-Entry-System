# Quick install (CPU-only torch — fine for serving, NOT for training):
#   .venv\Scripts\python.exe -m pip install -r requirements.txt
#
# ---------------------------------------------------------------------------
# GPU / CUDA — read this before training
# ---------------------------------------------------------------------------
# 1) Do you have an NVIDIA GPU? Run:  nvidia-smi
#    - Command not found / errors  -> no usable NVIDIA GPU: train on
#      Kaggle Notebooks (free ~30 GPU-hrs/week) or Colab instead.
#    - Shows a table -> note two things:
#        "CUDA Version: X.Y" (top right)  = the MAX cuda toolkit your
#                                           driver supports
#        Memory (e.g. 4096MiB)            = your VRAM budget
#
# 2) Install the CUDA build of torch. Plain `pip install torch` gives the
#    CPU build. Pick the wheel whose cuXXX is <= your driver's CUDA Version
#    (exact command generator: https://pytorch.org/get-started/locally):
#
#      .venv\Scripts\python.exe -m pip uninstall -y torch torchvision
#      .venv\Scripts\python.exe -m pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128
#
#    GOTCHA: if the CPU build is already installed, `--upgrade` will NOT
#    switch it — pip considers 2.x.x+cpu equal to 2.x.x+cu128. You must
#    uninstall first (as above) or add --force-reinstall.
#
# 3) Verify before training:
#      .venv\Scripts\python.exe -c "import torch; print(torch.cuda.is_available(), torch.cuda.get_device_name(0) if torch.cuda.is_available() else '')"
#    Must print True + your GPU name.
#
# 4) VRAM budget (training flags):
#      >= 8 GB : defaults are fine (train_yolo --imgsz 1024, train_unet --batch 8)
#      4-6 GB  : train_yolo --imgsz 640 ; train_unet --imgsz 512 --batch 2
#      OOM?    : halve --batch first, then --imgsz
# ============================================================================
